## Project Overview

Cross-platform adblocker browser extension ("adblocky") targeting **Chrome** (Manifest V3) and **Safari** (via Safari Web Extension). Uses WXT framework, React + Tailwind for UI, and TypeScript throughout. Blocks standard web ads via declarativeNetRequest (DNR) + cosmetic filtering, and streaming ads on YouTube, Twitch, Spotify, and Hulu via dedicated content scripts.

## Build & Development

```bash
npm run dev              # Dev mode with HMR (Chrome)
npm run dev:chrome       # Dev mode targeting Chrome
npm run build:chrome     # Production build → .output/chrome-mv3/
npm run build:safari     # Production build → .output/safari/
npm run build:filters    # Download filter lists → compile DNR + WebKit JSON
npm run build            # build:filters + wxt build (both platforms)
npm test                 # Run all tests (vitest)
npx vitest run path/to/test.ts  # Single test file
npm run typecheck        # TypeScript type checking
```

### Loading in Chrome
1. `npm run dev` (or `npm run build:chrome`)
2. `chrome://extensions` → Developer Mode → Load unpacked → `.output/chrome-mv3/`

## Architecture

**Framework:** WXT (wxt.dev) — auto-discovers entrypoints, generates manifest, handles multi-browser builds.

### Entrypoints (`src/entrypoints/`)
- `background.ts` — Service worker: DNR ruleset management, allowlist (dynamic rules), block stats, message hub
- `content.ts` — Global content script (all pages): cosmetic filtering via CSS injection + MutationObserver
- `youtube.content.ts` — YouTube ad skip, anti-adblock wall removal, injects `youtube-player.js` into MAIN world
- `twitch.content.ts` — M3U8 playlist ad stripping, GraphQL ad op blocking, mute-and-overlay fallback (MAIN world)
- `spotify.content.ts` — Audio ad detection/muting, ad UI hiding (MAIN world)
- `hulu.content.ts` — Ad break detection and UI cleanup
- `popup/` — React popup: global toggle, per-site allowlist, block counter
- `options/` — React options page: filter list management, streaming settings, allowlist editor

### Build-Time Filter Compiler (`scripts/`)
- `lists.config.ts` — Filter list URLs (EasyList, EasyPrivacy, uBlock, Peter Lowe's, Fanboy's)
- `parsers/abp-parser.ts` — ABP/uBO syntax → intermediate representation (network + cosmetic + scriptlet filters)
- `parsers/dnr-compiler.ts` — IR → Chrome DNR JSON (respects 28K rule / 900 regex caps per ruleset)
- `parsers/webkit-compiler.ts` — IR → Safari WebKit Content Blocker JSON (140K rule cap)
- `build-filters.ts` — Master script: downloads lists, runs both compilers, outputs to `rules/` + `safari-rules/` + `public/cosmetic-filters.json`

### Shared Libraries (`src/lib/`)
- `messaging.ts` — Typed message protocol (background ↔ content ↔ popup)
- `storage.ts` — Settings & block stats persistence via browser.storage.local
- `stats.ts` — Per-tab block counting (in-memory, service worker lifetime)

### Static Assets (`public/`)
- `youtube-player.js` — MAIN world script injected into YouTube pages (intercepts JSON.parse, fetch, ytInitialPlayerResponse to strip ad config)
- `cosmetic-filters.json` — Domain → CSS selector map (generated by build:filters)
- `rules/*.json` — Compiled DNR rulesets (generated by build:filters)

## Key Design Decisions

- **WXT framework** — auto-generates manifest from entrypoint files, handles MV3 service worker lifecycle
- **Build-time filter compilation** (à la uBlock Origin Lite) — filter lists compiled to DNR JSON at build time, not runtime
- **DNR for network blocking, content scripts for cosmetic** — DNR handles request blocking; content scripts handle element hiding and streaming ad manipulation
- **Streaming ads use MAIN world scripts** — YouTube/Twitch/Spotify scripts run in page context to intercept player APIs, fetch, and JSON.parse
- **YouTube multi-layer approach** — (1) strip ad config from player response, (2) detect `.ad-showing` class and skip, (3) click skip buttons, (4) remove anti-adblock walls
- **Twitch M3U8 manipulation** — intercept fetch for HLS playlists, strip `twitch-stitched-ad` segments
- **DNR rule budgets** — 28K safe rules per ruleset (headroom below 30K limit), 900 regex rules (below 1K limit), 50 enabled rulesets max

## Testing

Tests live in `tests/` with Vitest. Browser APIs are mocked in `tests/setup.ts`. Primary test coverage: ABP parser (syntax variants), DNR compiler (output format, limits, dedup), WebKit compiler.
